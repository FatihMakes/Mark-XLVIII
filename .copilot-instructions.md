# GitHub Copilot Instructions for MARK XXXIX

## Project Context

MARK XXXIX is a cross-platform personal AI assistant that integrates with Google Gemini API to provide voice-based control, screen analysis, file management, and autonomous task execution. The system is modular with specialized action handlers, an intelligent planning/execution agent, and persistent memory management.

## Key Architecture Principles

1. **Modular Design**: Each capability lives in its own action module under `actions/`
2. **Standard Interfaces**: All actions follow a consistent input/output pattern
3. **Centralized AI**: Gemini API in main.py handles planning and response generation
4. **Memory System**: Long-term context stored in `memory/long_term.json`
5. **Cross-Platform**: OS-specific code abstracted with conditional imports
6. **Error Recovery**: Failures handled with retry logic and error analysis

## Code Style & Standards

### Python Guidelines
- **Version**: Python 3.11+ required
- **Style**: PEP 8 compliant
- **Type Hints**: Always use type hints for function parameters and returns
- **Documentation**: Comprehensive docstrings following Google format

### Function Signatures for Actions
```python
def action_name(params: dict) -> dict:
    """Description of action."""
    return {
        "success": bool,
        "result": str,      # Success message/data
        "error": str        # Error message if failed
    }
```

### Error Handling Pattern
```python
try:
    # Implementation
    return {"success": True, "result": "..."}
except Exception as e:
    logger.error(f"Failed: {str(e)}", exc_info=True)
    return {"success": False, "error": str(e)}
```

## Common Tasks

### Adding a New Action Module

1. **Create file** in `actions/new_action.py`:
```python
"""Module description and purpose."""

import logging
from typing import Dict

logger = logging.getLogger(__name__)

def new_action(params: Dict) -> Dict:
    """
    Action description.
    
    Args:
        params: {"param_name": "value", ...}
    
    Returns:
        {"success": bool, "result": str, "error": str}
    """
    try:
        # Implementation
        return {"success": True, "result": "..."}
    except Exception as e:
        logger.error(f"Error: {e}", exc_info=True)
        return {"success": False, "error": str(e)}
```

2. **Register in main.py**:
   - Add import: `from actions.new_action import new_action`
   - Add to tool dispatcher in executor loop

3. **Add to planner prompt** (`core/prompt.txt`):
```
new_action
  param_name: type (required|optional)
  other_param: type (description)
```

4. **Document in API_REFERENCE.md**

### Modifying Memory System

Memory is stored in `long_term.json` with categories:
- `identity`: User info
- `preferences`: User preferences
- `projects`: Active projects
- `relationships`: Contacts
- `wishes`: Goals
- `notes`: General notes

**To update memory**:
```python
from memory.memory_manager import update_memory

update_memory("preferences", "key", "value")
```

**To load memory**:
```python
from memory.memory_manager import load_memory

memory = load_memory()
```

### Fixing Common Errors

**ModuleNotFoundError**:
- Install missing package: `pip install module_name`
- Check `requirements.txt` and `pyproject.toml`

**API Authentication Fails**:
- Verify `config/api_keys.json` exists and has valid key
- Get new key from https://aistudio.google.com/apikey

**Audio Issues**:
- Check microphone selected: `import sounddevice as sd; print(sd.query_devices())`
- Verify audio rates: SEND=16000Hz, RECEIVE=24000Hz

**Tool Not Executing**:
- Verify tool name matches in planner prompt exactly
- Check tool_input parameters are correct JSON
- Ensure action module is imported in main.py
- Review action module for exceptions

## Common Patterns to Reuse

### Logging Pattern
```python

```

### Try-Catch Pattern
```python
try:
    # operation
except SpecificException as e:
    logger.error(f"Specific error: {e}")
    return {"success": False, "error": str(e)}
except Exception as e:
    logger.error(f"Unexpected error: {e}", exc_info=True)
    return {"success": False, "error": str(e)}
```

### Path Handling Pattern
```python
from pathlib import Path
import sys

def get_base_dir():
    if getattr(sys, "frozen", False):
        return Path(sys.executable).parent
    return Path(__file__).resolve().parent

BASE_DIR = get_base_dir()
CONFIG_PATH = BASE_DIR / "config" / "file.json"
```

### API Key Loading Pattern
```python
import json
from pathlib import Path

def _get_api_key():
    config_path = Path(__file__).parent.parent / "config" / "api_keys.json"
    with open(config_path, "r") as f:
        return json.load(f)["gemini_api_key"]
```

## Quick Answers

**Q: How do I add a new action?**
A: Create `actions/name.py`, implement standard interface, register in main.py, add to planner prompt.

**Q: How do I store user preferences?**
A: Use `update_memory("preferences", "key", "value")`

**Q: How do I call an external API?**
A: Use `requests` library, handle rate limiting, return standard format.

**Q: How do I add a new tool to Gemini?**
A: Add function definition to planner prompt with name, description, and parameters.

**Q: How do I test my changes?**
A: Run `python main.py`, test with voice commands, check logs for errors.

**Q: Where do I configure the system?**
A: API keys in `config/api_keys.json`, prompts in `core/prompt.txt`, settings in module files.
